# Arcada v0.1
Generated by [Rojo](https://github.com/rojo-rbx/rojo) 7.6.1.

## Getting Started
To build the place from scratch, use:

```bash
rojo build -o "Arcada v0.1.rbxlx"
```

Next, open `Arcada v0.1.rbxlx` in Roblox Studio and start the Rojo server:

```bash
rojo serve
```

For more help, check out [the Rojo documentation](https://rojo.space/docs).

## Remote modules

`init.client.luau` loads protected modules from `rudevhd-glitch/test`. Add future
source modules to `src/client/Manifest.luau`; no change to the loader is required.
The protection build exports this bootstrap separately as `dist/Client.luau`,
ready to be hosted in a dedicated loader repository. Native Luau LocalScripts
are wrapped directly by Arcada Guard Security, while Lua 5.1-compatible modules
receive both Prometheus and Arcada Guard Security protection.

Build protected files with:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\Setup-Protection.ps1
$env:ARCADA_MASTER_SEED = "use-a-long-private-random-secret"
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\Protect-Modules.ps1 -Preset Compatible -Encryption ArcadaGuardSecurity -BuildId release-001
```

`ArcadaGuardSecurity` adds a per-build encrypted container around the Prometheus
output and emits each protected file as one physical line.
Its HMAC-SHA256 build seed depends on the private master seed and `BuildId`; the
master seed is never written to `dist`. The payload is split across three
shuffled byte shards and decoded with rolling state plus an integrity check.
Each protected module also carries two inert decoy descriptors with matching
shapes, independent byte shards, false rolling states and false checksums. Their
order and the opaque selector change per build. This increases output size to
roughly three times the single-container layer and is intended to slow static
or template-based analysis; it is not a cryptographic guarantee against a
client owner who can instrument the runtime.
Use a new `BuildId` for every release. By default a cryptographic nonce makes
every invocation different. For reproducible output, also supply the same
private seed and `-NonceHex 00112233445566778899aabbccddeeff`. Never reuse a
BuildId/nonce pair for different release contents. `-Encryption None` retains
the previous Prometheus-only output.

`Loader.luau` and `Client.luau` use compact protected containers so the public
entry point remains encrypted without forcing executors to parse multi-megabyte
Lua tables. Client then loads the protected modules from the manifest.

Upload the six generated files to the root of `rudevhd-glitch/testload`. The
loader reads `Manifest.luau`, which then loads `Home.luau` and `Farming.luau`.
`FarmingController.luau` is kept as native Luau during protection because it
uses Luau type syntax; when mounted with Rojo, the bootstrap requires the same
file as a child ModuleScript.

For a private repository, set a fine-grained read-only token before running the
exported script:

```luau
getgenv().ARCADA_GITHUB_TOKEN = "github_pat_..."
```

Never embed or distribute this token inside `init.client.luau`. A public,
read-only distribution repository is safer for scripts shared with other users.

Protection is obfuscation, not irreversible encryption. Client code can still be
captured at runtime. The build pipeline is based on Prometheus by Elias Oelschner:
https://github.com/prometheus-lua/Prometheus
